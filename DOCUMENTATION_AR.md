# توثيق DWAP Token على BSC

## نظرة عامة

**DWAP Token** هو توكن حوكمة لامركزي على شبكة Binance Smart Chain (BSC) مصمم لإدارة البنية التحتية اللامركزية للويب3.

## المميزات الرئيسية

### 1. **التوكن (DWAP_Token.sol)**
- **الإجمالي**: 1 مليار توكن (1B)
- **العرض الأولي**: 1 مليار توكن
- **المعايير**: BEP20 + ERC20Votes
- **الترقية**: قابل للترقية عبر UUPS Proxy

#### الوظائف الأساسية:
```solidity
// ملاك البروتوكول يمكنهم حرق من خزينة البروتوكول
ownerBurn(uint256 amount)

// المجتمع يمكنهم حرق من أرصدتهم الخاصة
communityBurn(uint256 amount)

// تفويض سلطة التصويت
delegate(address delegatee)

// إنشاء نقطة مرجعية للأرصدة
snapshot()
```

### 2. **التحكم في الحرق (DWAP_BurnController.sol)**
- إدارة منطقية لحرق التوكنات
- حدود يومية قابلة للتكوين
- تتبع إحصائيات الحرق
- نقل الملكية للـ DAO

#### الوظائف:
```solidity
// المجتمع يحرق عبر الكنترولر
burnTokens(uint256 amount)

// المالك يحرق مباشرة
ownerBurnTokens(uint256 amount)

// تحديث سياسة الحرق
setBurnPolicy(bool enabled, uint256 dailyLimit, uint256 minAmount)
```

### 3. **الحوكمة (DWAP_Governor.sol)**
تطبيق OpenZeppelin Governor مع التالي:

| المعامل | القيمة | الوصف |
|---------|--------|-------|
| **تأخير التصويت** | 48 ساعة | وقت قبل فتح التصويت |
| **فترة التصويت** | 1 أسبوع | المدة المسموحة للتصويت |
| **نسبة النصاب** | 4% | النسبة المئوية المطلوبة للمشاركة |
| **حد الاقتراح** | 1 DWAP | الحد الأدنى للاقتراح |
| **قاعدة القبول** | 50%+ | التصويت بالأغلبية |

### 4. **Timelock (DWAP_Timelock.sol)**
- تأخير 2 يوم (172800 ثانية) قبل التنفيذ
- منع التنفيذ الفوري للمقترحات
- السماح للمجتمع بمراجعة والرد على القرارات

## سير عمل مثالي

### المرحلة 1: الإطلاق الأولي
1. نشر DWAP Token (كل التوكنات للمالك الأولي)
2. نشر DWAP Governor و Timelock
3. إعداد الأدوار والصلاحيات

### المرحلة 2: توزيع التوكنات
1. توزيع التوكنات للمجتمع (عبر liquidity pools, airdrops, etc)
2. المجتمع يفوض أصواتهم
3. المجتمع يحرق أي توكنات غير ضرورية

### المرحلة 3: نقل الملكية للـ DAO
```solidity
// إنشاء مقترح بـ Governor
governor.propose(
  [dwapTokenAddress],
  [0],
  [encodeCall(dwapToken.transferOwnership, governorAddress)],
  "Transfer DWAP ownership to DAO Governor"
)

// التصويت لمدة 7 أيام (مع 4% نصاب و 50%+ موافقة)

// انتظار 2 يوم (Timelock)

// تنفيذ المقترح
```

بعد هذا، كل التحكم بالتوكن يصبح من خلال الحوكمة اللامركزية!

### المرحلة 4: الحرق المجتمعي
```solidity
// أي عضو مجتمع يمكنه حرق التوكنات الخاصة به
dwapToken.communityBurn(amount)

// أو عبر DWAP Burn Controller
burnController.burnTokens(amount)
```

## الأسئلة الشائعة

**س: هل يمكن تجميد التوكنات؟**
ج: لا، بعد نقل الملكية للـ DAO، لا يمكن لأحد تجميد التوكنات إلا من خلال مقترح حوكمة.

**س: كم عدد الأشخاص المطلوب لإقرار مقترح؟**
ج: 4% من إجمالي التوكنات في التصويت + 50% موافقة من المصوتين.

**س: هل يمكن تقليل إجمالي العرض؟**
ج: نعم، من خلال الحرق (Owner أو Community)، لكن لا يمكن زيادته أبداً.

**س: متى ينتقل التحكم للـ DAO؟**
ج: عندما يصوت المجتمع على نقل الملكية للـ Governor.
